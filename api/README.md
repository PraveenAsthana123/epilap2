# Epilepsy Intelligence Platform — API

A thin, runnable FastAPI service over the platform artefacts: roles, the
seizure/epilepsy scenario catalogue, and the weighted severity **scoring** model.
Stateless and file/DB-backed, so it runs from a clean checkout.

## Run

```bash
cd api
pip install -r requirements.txt
python ../db/build_sqlite.py          # materialise db/epilepsy.db (roles, scenarios, EP001)
uvicorn main:app --reload             # http://127.0.0.1:8000
```

Interactive OpenAPI/Swagger docs: **http://127.0.0.1:8000/docs**

## Endpoints

| Method | Path | Purpose |
|---|---|---|
| GET | `/health` | Liveness check |
| GET | `/roles` | 9 roles + composite-score domain weights (sum = 1.0) |
| GET | `/severity` | The 4-level severity ladder |
| GET | `/scenarios` | Full scenario catalogue (optional `?category=`) |
| GET | `/scenarios/{id}` | One scenario (e.g. `SZ-F02`) |
| POST | `/score` | Weighted severity score from submitted answers |
| GET | `/patient/{id}` | Patient record + stored composite (e.g. `EP001`) |

### `POST /score` example

```json
{
  "role": "neurologist",
  "sections": [
    {"name": "seizure", "items": [{"level": 3, "weight": 1.0}, {"level": 4, "weight": 1.2}]},
    {"name": "aura",    "items": [{"level": 2, "weight": 0.7}]}
  ]
}
```

Returns per-section scores, the role score, and the severity band. Scoring is
identical to the viewer and `docs/scenarios/scoring-weightage.md`:
`section = Σ(level·weight)/Σ(weight)` → `role = mean(sections)` → band
(`<1.75 Mild · <2.5 Moderate · <3.25 Severe · else Refractory/Status`).

## Tests

```bash
cd api && python -m pytest -q        # positive + negative (validation, 404, weighting)
```

## Notes
- Data sources: `data/analysis/epilepsy_scenarios.csv`, `data/analysis/domain_weightage.csv`,
  `db/epilepsy.db` — all regenerated by `analysis/build_scenarios.py` and `db/build_sqlite.py`.
- Input validation is enforced by Pydantic (severity level constrained to 1–4 → `422` on violation).
